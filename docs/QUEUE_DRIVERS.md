# Порівняння драйверів черг у Laravel

## Таблиця порівняння

| Драйвер | Де зберігаються jobs | Потрібен worker? | Коли використовувати            |
|---------|---------------------|------------------|---------------------------------|
| **sync** | Ніде (виконується одразу) | Ні | Локальна розробка, тести        |
| **database** | Таблиці `jobs`, `failed_jobs` | Так | Невеликі проекти, без Redis     |
| **redis** | Redis (списки) | Так | Production, високе навантаження |
| **sqs** | AWS SQS | Так | Великі проекти на AWS           |
| **beanstalkd** | Beanstalkd сервер | Так | Класичні черги                  |

---

## Детальний опис драйверів

### 1. sync (синхронний)

```env
QUEUE_CONNECTION=sync
```

**Як працює:**
- Job виконується одразу в тому ж процесі
- Не зберігається в чергу
- Worker не потрібен

**Плюси:**
- ✅ Не потрібен worker
- ✅ Легкий дебаг (задачі виконуються одразу)
- ✅ Не потребує додаткових сервісів

**Мінуси:**
- ❌ Задачі блокують користувача
- ❌ Не підходить для production

**Коли використовувати:**
- Локальна розробка
- Написання тестів
- Швидке налагодження (debug)

---

### 2. database

```env
QUEUE_CONNECTION=database
```

**Як працює:**
- Job серіалізується і зберігається в таблицю `jobs`
- Worker забирає задачі з БД
- Після виконання задача видаляється

**Плюси:**
- ✅ Не потребує Redis
- ✅ Легко налаштувати (одна таблиця)
- ✅ Задачі не блокують користувача

**Мінуси:**
- ❌ Навантаження на базу даних
- ❌ Повільніше за Redis
- ❌ Не підходить для високого навантаження

**Коли використовувати:**
- Навчальні проекти
- Staging середовище
- Невеликі проекти без Redis

---

### 3. redis

```env
QUEUE_CONNECTION=redis
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_PASSWORD=null
```

**Як працює:**
- Job зберігається в Redis (структура даних - список)
- Worker забирає задачі з Redis
- Дуже швидко (in-memory)

**Плюси:**
- ✅ Дуже швидко (in-memory)
- ✅ Добре масштабується
- ✅ Підтримує різні черги (queues)

**Мінуси:**
- ❌ Потребує Redis сервер
- ❌ Складніше налаштування

**Коли використовувати:**
- Production середовище
- Високе навантаження
- Коли Redis вже використовується для кешу

---

### 4. sqs (Amazon Simple Queue Service)

```env
QUEUE_CONNECTION=sqs
SQS_KEY=your-aws-key
SQS_SECRET=your-aws-secret
SQS_PREFIX=https://sqs.us-east-1.amazonaws.com/your-account-id
SQS_QUEUE=your-queue-name
SQS_REGION=us-east-1
```

**Як працює:**
- Job зберігається в AWS SQS
- Worker забирає задачі через API

**Коли використовувати:**
- Великі проекти на AWS
- Розподілені системи
- Потрібна висока надійність

---

### 5. beanstalkd

```env
QUEUE_CONNECTION=beanstalkd
BEANSTALKD_HOST=127.0.0.1
BEANSTALKD_PORT=11300
```

**Коли використовувати:**
- Класичні черги
- Легковагові проекти

## Важливе зауваження

> **Код Job не змінюється при перемиканні драйвера!**

```php
// Цей код працює однаково для ВСІХ драйверів
class SendWelcomeEmail implements ShouldQueue
{
    public function handle(): void
    {
        Mail::to($this->client->email)->send(new WelcomeMail($this->client));
    }
}

// Відправка в чергу - однакова для всіх
SendWelcomeEmail::dispatch($client);
```

Можна вільно змінювати `QUEUE_CONNECTION` в `.env`:
- `sync` → `database` → `redis` → `sqs`

Код залишається той самий. Змінюється лише місце зберігання та спосіб обробки.

## Рекомендації по середовищах

| Середовище | Рекомендований драйвер | Чому |
|------------|----------------------|------|
| **Local (розробка)** | `sync` або `database` | Швидкий дебаг, не треба worker |
| **Testing (тести)** | `sync` | Задачі виконуються одразу |
| **Staging (тестовий сервер)** | `database` | Легко налаштувати, без Redis |
| **Production (продакшен)** | `redis` | Швидко, масштабується |

---

## Як перемикатися між драйверами

### На sync (локальна розробка)

```env
QUEUE_CONNECTION=sync
```

### На database (тестування черг)

```env
QUEUE_CONNECTION=database
php artisan queue:work database
```

### На redis (продакшен)

```env
QUEUE_CONNECTION=redis
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
php artisan queue:work redis --sleep=3 --tries=3
```

## Перевірка який драйвер використовується

```bash
php artisan tinker
>>> config('queue.default')
=> "database"

>>> exit
```

## Порівняння продуктивності

| Драйвер | Час виконання 1000 jobs | Навантаження |
|---------|------------------------|--------------|
| **sync** | ~0.5 сек | На PHP процес |
| **database** | ~5-10 сек | На БД |
| **redis** | ~1-2 сек | На RAM |

---

## Висновок

| Драйвер | Для чого |
|---------|----------|
| **sync** | Локальна розробка, тести |
| **database** | Staging, невеликі проекти |
| **redis** | Production, високе навантаження |

**Головне правило:**
- Код Job не змінюється
- Змінюється тільки `.env`
- Для production обов'язково використовувати worker
