## 4. Корінна причина

**Що призвело до інциденту?** (на основі логів, Sentry, метрик)

**Приклад:**

При деплої release 2.1.0 змінили логіку розрахунку комісії:

```php
// До: комісія тільки для сум > 10000
if ($amount > 10000) {
    $commission = $amount * 0.005;
}

// Після: комісія для всіх сум > 0 (помилка)
if ($amount > 0) {
    $commission = $amount * 0.005;
}
```

Це призвело до того, що для переказів на невеликі суми (100-1000 UAH) додавалась комісія, якої не мало бути, і баланс рахунку ставав від'ємним.

### Докази з логів

```kql
correlation_id: "83a74f66-2847-4c87-b20a-651947a70529"
```

**Контекст помилки:**
- `account_from_id`: 1
- `amount`: 100.00
- `balance`: 50.00
- `commission`: 0.50
- `total_deduct`: 100.50

**Висновок:** Балансу (50.00) не вистачало на суму + комісію (100.50).
